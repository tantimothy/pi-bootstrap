📌 Post-Install Notes:

  🔑 Pi-hole: change/reset the admin web UI password
     docker exec -it pihole pihole setpassword

  🌐 Pi-hole: config lives at ./etc-pihole/pihole.toml on the host.
     Edits via env vars only seed it on first creation — use the web UI
     (Settings > All Settings) or 'pihole-FTL --config <key> <value>' afterward.

  ➕ WireGuard: add a client / device
     Open the Web Dashboard above, click 'New Client', then scan the QR code
     with the WireGuard mobile app, or download the .conf for desktop clients.

  📡 WireGuard: remote access requires router port forwarding
     Forward external UDP port ${WG_PORT} to this Pi's local static IP
     on your home router/gateway, or remote peers will never reach the tunnel.

  👀 WireGuard: check connected peers / tunnel status
     docker exec -it wg-easy wg show

  💾 WireGuard: persistent state (server keys, peer configs) lives at
     ./etc-wireguard on the host — back this up; losing it invalidates all
     existing client configs and requires re-adding every device.

  🔑 WireGuard: change the dashboard login password
     1. docker run --rm -it ghcr.io/wg-easy/wg-easy wgpw 'your_new_password'
     2. Copy the printed hash into PASSWORD_HASH= in .env (single-quote it,
        e.g. PASSWORD_HASH='$2y$12$...' — unquoted $ characters get
        mangled when this script sources .env)
     3. docker compose up -d --force-recreate wg-easy

  📊 Grafana: pre-provisioned dashboards are in the 'Pi Network' folder.
     If dashboards are missing (no internet at deploy time), import manually:
     Pi-hole → Dashboards > Import > ID 10176
     WireGuard → Dashboards > Import > ID 12177

  ⚙️  FTLCONF_webserver_api_password (Pi-hole) only seeds pihole.toml on FIRST
     container creation, then is ignored — use 'pihole setpassword' to
     change it later, not the .env value.
  ⚙️  WG_HOST (WireGuard) is read fresh on every container start, so editing
     it in .env + 'docker compose up -d --force-recreate wg-easy' does take
     effect — but any client .conf/QR code you already downloaded is a
     static snapshot of the OLD host and won't update; redownload it from
     the dashboard for each existing peer after changing WG_HOST.
